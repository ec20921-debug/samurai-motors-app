# Samurai Motors 管理者向け作業手順書

**対象者**: 日本側管理メンバー（飯泉・鈴木・五木田・Daisuke）

**最終更新**: 2026年4月20日

**対応システム**: v7.3（顧客対応系）+ v7-operations（勤務系）

---

## はじめに

本手順書は、Samurai Motorsの予約管理システムを日本から運用するための操作手順をまとめたものです。日常業務で発生頻度の高い順に記載しています。特に決済確認フローは最重要業務ですので、必ず熟読してください。

---

## 1. 日常の流れ

管理者の1日は以下のサイクルで回ります。

朝はTelegram管理グループを開き、新しい予約トピックが作成されていないか確認します。タスクトピックには当日のタスク一覧が届いています（JST 8:00自動配信）。勤怠ログトピックにはロンさんの出勤通知がGPS付きで届きます。

日中は予約が入るたびにトピックが自動作成され通知されます。作業開始・Before写真・作業完了・After写真の通知が順次届きます。顧客からのメッセージもトピックに転送されます。

夕方以降は決済スクリーンショットの確認とシート更新を行います。日報トピックにはJST 20:00に自動サマリーが届きます。

---

## 2. 予約が入ったときの確認

予約が入ると、管理グループに自動でトピックが作成されます。

**手順**

1. Telegramで「Samurai Motors 管理マスター」グループを開きます
2. 新しいトピック（例：「🚗 BK-0042 Hisanori / Camry」）を確認します
3. トピックを開き、以下の情報を確認します
   - 予約日時
   - プラン名
   - 料金内訳（基本料金 + 出張料 = 合計）
   - 場所（Googleマップリンク付き）

予約確定後、顧客には駐車場所のヒアリング（車両前面写真と駐車階数の確認）が自動送信されます。顧客から写真が届くとトピック内に転送されます。

---

## 3. 顧客への返信方法

顧客からメッセージが届くと、該当トピックに自動転送されます。

**手順**

1. トピック内に表示された顧客メッセージの下にある「💬 返信」ボタンをタップします
2. Botから「返信メッセージを入力してください」と表示されます
3. テキスト、写真、動画のいずれかで返信を入力し送信します
4. 予約Bot経由で顧客に届きます

「💬 返信」ボタンを押してから5分以内に返信してください。5分経過すると返信モードが解除されます。その場合は再度ボタンを押してやり直してください。

---

## 4. 決済確認フロー

### ⚠️⚠️⚠️ 最重要：この手順を怠ると顧客に催促メッセージが送信され続けます ⚠️⚠️⚠️

作業完了後、QR決済コードが顧客に自動送信されます。顧客が支払いを完了すると、決済スクリーンショットがトピックに転送されます。

**確認・更新手順**

1. トピックに届いた決済スクリーンショットを確認します
2. Googleスプレッドシートの「BOOKINGS」シートを開きます
3. 該当する予約番号（BK-XXXX）の行を見つけます
4. T列「payment_status」のセルをクリックします
5. プルダウンから選択します
   - **「清算済み」** → 金額が一致している場合。顧客にお礼メッセージが自動送信されます
   - **「要確認」** → 金額が違う、画像が不鮮明等の場合。催促は停止されますが、個別に顧客への確認が必要です

### なぜこの手順が重要か

payment_statusが「QR送信済み」のままだと、作業完了から24時間後に顧客へ催促メッセージが自動送信されます。以降も24時間ごとに繰り返されます。催促が3回以上になると管理トピックに🚨赤色アラートが表示されます。その場合は電話やメッセージで直接顧客に連絡してください。

### 料金の確認ポイント

v7.3から料金はDelivery fee（出張料）込みの合計金額で表示・請求されています。例えばKIYOME（セダン）の場合、基本料金$12 + 出張料$2 = 合計$14が請求額です。スクリーンショットの金額がこの合計と一致しているか確認してください。

## 5. 作業進行の確認

洗車作業の各ステップで、トピックに通知が届きます。

**通知の流れ**

作業開始の通知として「▶ 作業開始 - BK-XXXX 顧客名 / 車種 スタッフ：ロン 開始：10:30」のようなメッセージが届きます。続いてBefore写真4枚（前・後・左・右）がまとめて転送されます。作業完了の通知として「✅ 作業完了 - BK-XXXX 所要：35分」のようなメッセージが届きます。After写真4枚が転送され、その後QR送信済みの通知が届きます。

管理者は特に操作不要です。作業の進行状況を確認するだけで構いません。

## 6. QRコードの切り替え

銀行口座の変更やキャンペーン時にQRコードを切り替える手順です。

**手順**

1. 新しいQRコード画像をGoogleドライブの「SamuraiMotors_QRCodes」フォルダにアップロードします
2. アップロードした画像を右クリックし「リンクを取得」からリンクをコピーします
3. スプレッドシートの「QR_CODES」シートを開きます
4. 新しい行を追加し、qr_id、image_url（コピーしたリンク）、descriptionを入力します
5. 新しい行のactive列を「TRUE」にします
6. 旧QRコードの行のactive列を「FALSE」にします

active = TRUEの行は常に1行だけにしてください。複数あるとエラーの原因になります。

## 7. 料金・営業時間の変更

料金や営業時間の変更は、スプレッドシートの「料金設定」シート（Plan_Prices）を編集するだけで反映されます。コードの変更は不要です。

**変更可能な項目**

プランごとの基本料金（セダン/SUV）、プランごとの所要時間（セダン/SUV）、出張料（Delivery fee）の金額、営業開始時刻・営業終了時刻、移動バッファ時間。

シートを更新すると、最大60秒後に予約ミニアプリ、カレンダー空き枠検索、QR決済の請求金額、BOOKINGSシートへの記録に自動反映されます。

## 8. タスク管理

現場スタッフや日本側メンバーへのタスクを登録できます。

**「新規タスク」シートでの入力手順**

1. v7-operationsスプレッドシートの「新規タスク」シートを開きます
2. A列「期限」に日付ピッカーで日付を選択します
3. B列「担当者」のプルダウンから担当者を選択します（ロン / 飯泉 / 鈴木 / 五木田）
4. C列「タスク内容」にタスクの内容を入力します
5. D列「繰返し」のプルダウンから繰返しルールを選択します（なし / 毎日 / 毎週月曜 / 毎週金曜 / 毎月1日 / 毎月10日 / 毎月末）
6. E列「優先度」のプルダウンから優先度を選択します（通常 / 高 / 重要）
7. F列の「追加」ボタンをクリックします

タスクはTasksシートに自動転記されます。

**通知タイミング**

現場スタッフ（ロン）へは毎朝プノンペン時間8:00に勤務BotのDMで届きます。日本側メンバーへは毎朝日本時間8:00に管理グループのタスクトピックに届きます。担当者ごとにグルーピングされた形で表示されます。

**登録済みの定期タスク**

ロンさんへ給与支払いは毎月10日に飯泉さんへ通知されます。オフィス賃料$750の支払いは毎月1日に飯泉さんへ通知されます（5日までに必須、7日超過で$20/日ペナルティ）。携帯電話代$4をロンさんへ渡す件は毎月10日に飯泉さんへ通知されます。

## 9. 経営ダッシュボードの確認

スプレッドシート上で売上・顧客・稼働状況を可視化できます。

**表示手順**

1. v7スプレッドシート（予約管理用）を開きます
2. メニューバーの「📊 ダッシュボード」をクリックします
3. 「📊 ダッシュボード更新」を選択します
4. 「📊 経営ダッシュボード」シートが更新・表示されます

期間はダッシュボード上部のプルダウンで切り替えられます（今月 / 先月 / 今年 / 昨年 / 過去12ヶ月）。毎朝JST 7:30に自動更新されますが、最新データを見たい場合は手動で更新してください。

**確認できる情報**

KPIカード6枚として、売上合計・予約件数・平均単価・新規顧客数・リピート率・未回収金額が表示されます。月次推移グラフ、プラン別売上構成の円グラフ、決済状態別の円グラフ、新規vsリピートの円グラフ、日別の売上棒グラフ、プラン別売上ランキング、顧客別LTVトップ10、経営アラート（未入金48h超等）も確認できます。

## 10. 勤怠・日報の確認

**勤怠**

管理グループの「勤怠ログ」トピックに、出勤・退勤のたびに通知されます。通知にはGPS位置情報のマップリンクが付いており、タップすると打刻場所を確認できます。詳細はv7-operationsスプレッドシートの「勤怠記録」シートで確認できます。

**日報**

毎日JST 20:00に管理グループの「日報」トピックに自動サマリーが投稿されます。詳細はv7-operationsスプレッドシートの「日報」シートで確認できます。

---

## 11. 定休日・祝日について

日曜日は定休日です。カンボジアの公式祝日も定休日として設定されています。これらの日は予約ミニアプリ上で「Closed」「Holiday」と表示され、顧客が予約を入れることはできません。

祝日の追加や営業日の変更が必要な場合は、BookingLogic.gsのCAMBODIA_HOLIDAYS配列とbooking.htmlのCAMBODIA_HOLIDAYSオブジェクトの両方を更新する必要があります。Claude Codeに依頼してください。

## 12. トラブル時の対応

### Botが応答しなくなった場合

1. GASエディタを開きます（v7スプレッドシート → 拡張機能 → Apps Script）
2. 左側のメニューから「トリガー」（時計アイコン）を開きます
3. トリガーが3つ存在するか確認します
   - pollTelegramUpdates（1分間隔）
   - processTelegramQueue（1分間隔）
   - cleanupOldProcessedIds（1時間間隔）
4. トリガーがない場合、関数セレクタで「setupV7Triggers」を選んで▶実行します

### 顧客への通知が届かない場合

管理グループのチャットログを削除した場合、トピックが消失して通知が届かなくなることがあります。システムは自動復旧を試みますが、完全にリセットしたい場合は、GASエディタで「resetAllCustomerTopicIds」を実行してください。次回の顧客メッセージまたは予約時に新しいトピックが自動作成されます。

### 予約可能日に空き枠が表示されない場合

Googleカレンダーに終日イベント（出張予定等）が入っていると、その日は全枠がブロックされます。カレンダーを確認し、不要なイベントがあれば削除してください。予約ミニアプリの空き枠画面には診断情報が表示されており、何がブロックの原因になっているか確認できます。

### 緊急停止

Botの動作を完全に停止したい場合は、GASエディタで「resetV7」を実行してください。全トリガーが削除されBotが停止します。再開時は「setupV7Triggers」を実行してください。

### キューが詰まった場合

メッセージが処理されず溜まっている場合は、「clearTelegramQueue」を実行してください。未処理のメッセージは破棄されます。
