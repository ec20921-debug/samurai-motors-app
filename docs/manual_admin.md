# Samurai Motors 業務管理システム - Admin向けマニュアル

## はじめに

このマニュアルは、Samurai Motors 業務管理システムの管理者（Admin）向けの操作ガイドです。
Telegramの「Admin サムライモーターズ業務管理」グループから各機能にアクセスできます。

---

## アクセス方法

### 方法1: ボットのメニューボタンから
1. Telegramで「**Samuraimotors 業務管理**」ボットを開く
2. 左下の **「業務管理」ボタン** をタップ
3. ホーム画面が開きます

### 方法2: Adminグループのリンクから
1. Adminグループで `/menu` と送信（またはピン留めメッセージのリンクをタップ）
2. 表示されたリンクから各機能に直接アクセスできます

![ホーム画面](screenshots/home.png)

## 1. タスク管理

現場スタッフへの作業指示を作成・管理する機能です。

![タスク管理画面](screenshots/task-manager.png)

### タスクの確認

1. ホーム画面で **「タスク管理」** をタップ
2. 一覧画面に未完了タスクと完了タスクが表示されます
3. 各タスクカードには以下の情報が表示されます：
   - 担当者名
   - 期限
   - タスク内容
   - ステータス（未着手 / 進行中 / 完了 / 未完了）

### タスクの新規作成

![タスク作成画面](screenshots/task-create.png)

1. **「＋ 新規タスク作成」** ボタンをタップ
2. 以下を入力：
   - **担当者**: ドロップダウンから選択（「その他」で自由入力も可）
   - **期限**: 日付を選択
   - **やるべきこと**: タスクの内容を記入
3. **「タスクを作成」** ボタンをタップ
4. 作成完了すると、担当スタッフのTelegramに通知が届きます（クメール語に自動翻訳）

### タスクの編集

1. タスクカードの **「✏️ 編集」** ボタンをタップ
2. 編集モーダルで担当者・期限・内容を修正
3. **「保存」** をタップ

### タスクの進捗管理

- 毎朝カンボジア時間8:00に、スタッフに未完了タスクの通知が届きます
- スタッフは通知のボタンで **「✅ 完了」** または **「❌ 未完了」** を報告します
- 未完了の場合は理由も入力されます
- タスクの状態はスプレッドシートの「Tasks」シートに記録されます

## 2. 経費管理

業務経費の記録・管理を行う機能です。AdminはField Staffより多くの管理項目を利用できます。

![経費入力画面](screenshots/expense-entry.png)

### 経費の登録

1. ホーム画面で **「経費入力」** をタップ
2. レシートの写真を撮影（推奨）
3. 以下を入力：
   - **取引日**: お金を使った日付
   - **金額**: 金額を入力
   - **通貨**: USD / KHR / JPY から選択
   - **お店の名前**: 購入先
   - **何を買った？**: 購入内容
   - **登録者**: 自分の名前を選択

### Admin専用項目

Adminとしてアクセスした場合、以下の追加項目が表示されます：

- **勘定科目**: 消耗品費、水道光熱費、交通費 など（freee連携用）
- **経費スプレッドシートを開く**: スプレッドシートの「Expenses」シートを直接開くリンク

### 経費データの確認

- 「履歴」タブで過去の経費一覧を確認できます
- スプレッドシートの「Expenses」シートで全経費データを閲覧・編集できます
- freeeへの連携が必要な場合は、勘定科目列を参考にしてください

## 3. 洗車登録（ジョブ管理）

洗車作業の記録を管理する機能です。現場スタッフが登録した内容がAdminグループにも通知されます。

![洗車登録画面](screenshots/job-manager.png)

### 登録内容の確認

洗車が登録されると、Adminグループに以下の情報が通知されます：
- 顧客名・電話番号
- 場所（建物名・部屋番号）
- 車両情報（車種・ナンバー）
- 選択プラン
- 作業前・作業後の写真
- 所要時間

### データの管理

- スプレッドシートの「Jobs」シートで全洗車記録を確認できます
- 顧客情報は「Customers」シートに蓄積されます

## 4. 勤怠管理

スタッフの出退勤を管理する機能です。

![勤怠打刻画面](screenshots/attendance.png)

### 勤怠データの確認

- スプレッドシートの「Attendance」シートで全スタッフの出退勤記録を確認できます
- 記録される項目：
  - スタッフ名
  - 出勤日時
  - 退勤日時
  - 勤務時間（自動計算）

### ポイント
- 二重打刻はシステムが自動防止します
- 打刻忘れなどの修正はスプレッドシートから直接編集できます

## 6. 前日レポート（自動通知）

毎朝 **日本時間 9:00**（カンボジア時間 7:00）にAdminグループへ自動送信されます。
**前日分**の業務結果をまとめたレポートです。朝一でこの通知を見れば、前日の状況と今やるべきことが把握できます。

### レポートの内容

1. **洗車実績**: 前日の洗車件数・顧客一覧・プラン内訳
2. **勤怠状況**: 各スタッフの出勤・退勤時間・勤務時間
3. **日報**: スタッフからの業務報告
4. **経費**: 前日の経費件数・合計金額
5. **タスク状況**: 完了/未完了件数・期限超過タスクの警告・未完了タスク一覧

### レポートが届かない場合

以下を確認してください：
1. Google Apps Scriptのトリガーが設定されているか
   - Apps Scriptエディタ → トリガー（時計アイコン）
   - `sendDailySummary` が UTC 0:00 に設定されているか確認
2. トリガーを再設定する場合は `setupV5Triggers()` を実行

## 7. システム管理

### スタッフの追加

新しいスタッフを追加する場合：
1. Apps Scriptエディタで `STAFF_REGISTRY` にスタッフ情報を追加
2. Chat IDはTelegramのユーザーIDを使用

```
STAFF_REGISTRY = {
  'チャットID': { name: 'スタッフ名', role: 'field' }
}
```

3. 保存 → 新しいバージョンでデプロイ

### トリガーの確認

| 機能 | UTC | 日本時間 | カンボジア時間 |
|------|-----|---------|--------------|
| 前日レポート（Admin向け） | 0:00 | 9:00 | 7:00 |
| 繰返しタスク生成 | 0:00 | 9:00 | 7:00 |
| 朝タスク通知（スタッフ向け） | 1:00 | 10:00 | 8:00 |

### スプレッドシートの構成

| シート名 | 内容 |
|----------|------|
| Jobs | 洗車作業記録 |
| Customers | 顧客情報 |
| Tasks | タスク管理 |
| Expenses | 経費記録 |
| DailyReports | 日報 |
| Attendance | 勤怠記録 |

### デプロイの更新手順

コードを変更した場合：
1. Apps Scriptエディタでコードを更新・保存
2. **「デプロイ」→「デプロイを管理」**
3. 鉛筆アイコン → バージョンを **「新バージョン」** に変更
4. **「デプロイ」** をクリック
5. URLは変わりません（既存の連携はそのまま動作します）

## よくある質問

**Q: 前日レポートが届かない**
A: Apps Scriptのトリガー画面で `sendDailySummary` の実行状況を確認してください。エラーがある場合は赤いマークが表示されます。`setupV5Triggers()` を再実行すると復旧できます。毎朝JST 9:00（UTC 0:00）に前日分が届きます。

**Q: スタッフの入力を修正したい**
A: スプレッドシートから直接該当のセルを編集できます。

**Q: 新しい機能を追加したい**
A: Apps Scriptのコードを編集し、新しいバージョンでデプロイしてください。HTMLファイルはGitHub Pagesで管理しています。

**Q: ボットが反応しない**
A: Apps Scriptエディタの「実行数」でエラーログを確認してください。Webhookの再設定が必要な場合は `setupWebhook()` を実行してください。

**Q: 経費の勘定科目が表示されない**
A: 勘定科目はAdmin専用項目です。AdminのTelegram IDが正しく設定されているか確認してください。
