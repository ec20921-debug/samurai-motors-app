# パートナープログラム セットアップ手順書

本ドキュメントは Samurai Motors **パートナープログラム（紹介制度）** の自動化フローを
Google Apps Script + Google Form + 既存の v7-operations スプレッドシートで運用するための
**管理者向けセットアップ手順** です。

---

## 🎯 ゴール

- 提携したい経営者・起業家に Google Form の URL を送る
- 回答すると自動で「パートナー」シートに登録される
- 管理グループ（Telegram）に **✅承認 / ❌却下** ボタン付きの通知が届く
- 承認をタップすれば Welcome DM（パートナーID・紹介コード・無料体験コード）が自動送信

---

## 📋 全体フロー

```
① パートナー候補: Google Form に回答
        ↓
② GAS onFormSubmit: 「パートナー」シートへ追記 + ID/コード採番
        ↓
③ GAS: Telegram 管理グループ「パートナー」トピックへ
        ┌ 承認ボタン付きメッセージ送信
        ↓
④ Admin: ✅承認 をタップ
        ↓
⑤ GAS: 契約期間(6ヶ月)設定 + Welcome DM 自動送信
        ↓
⑥ パートナー: 紹介コード・体験コードを受け取り、紹介活動開始
        ↓
⑦ 予約Bot: 予約時にコード入力 → 紹介履歴シートに記録
        ↓
⑧ 月末: 紹介履歴から支払額を集計 → ABA 送金
```

---

## 🛠 セットアップ手順

### STEP 1. GAS にコードを反映

v7-operations GAS プロジェクトに以下のファイルを貼り付け or 上書き：

| ファイル | 変更内容 |
|---|---|
| `Config.gs` | `ADMIN_PARTNER_THREAD_ID` `PARTNER_WELCOME_KIT_URL` `PARTNER_TRIAL_VOUCHER_PREFIX` を追加 |
| `PartnerManager.gs` | **新規ファイル** |
| `Setup.gs` | `setupPartnerProgram()` `setupPartnerFormTrigger()` を追記 |
| `Router.gs` | `partner_lookup_by_code` `partner_record_referral` を追加 |
| `TaskManager.gs` | `handleTaskCallback_` にパートナーコールバック振分を追加 |

反映後、一度「保存」してください。

### STEP 2. 設定シートを作成

GAS エディタで以下の関数を順に実行：

```
setupPartnerProgram()
```

実行すると v7-ops スプレッドシートに2枚のシートが追加されます：
- **パートナー**（申請情報・紹介コード・契約状況）
- **紹介履歴**（コミッション集計用）

### STEP 3. Telegram パートナートピックを作成（任意）

管理グループ（フォーラム型）で **「パートナー」** という名前のトピックを作成してください。
作成しない場合は「タスク」トピックに合流します（動作は問題ありません）。

トピック URL から thread_id を取得し、Script Properties に登録：

```
キー:   ADMIN_PARTNER_THREAD_ID
値:    （URL の真ん中の数字。例: t.me/c/3856480475/200/350 → 200）
```

### STEP 4. Welcome Kit PDF URL を登録（任意）

Welcome Kit（紹介マニュアル・Q&A・ブランド素材・QR画像）を Google Drive または自社サイトで
公開 → URL を以下のキーに登録：

```
キー:   PARTNER_WELCOME_KIT_URL
値:    https://...（任意）
```

未登録の場合は Welcome DM から「📎 Welcome Kit: ...」行が省略されます。

### STEP 5. Google Form を作成

#### 5-1. 新規フォーム作成

Google Drive → 新規 → Google Form → **「Samurai Motors パートナー申請」**

#### 5-2. 質問を追加（ラベル完全一致で作成）

⚠️ **ラベルは以下と1文字違わず完全一致**させてください（GAS の `PARTNER_FORM_FIELDS_` と照合）。
コピペ推奨。

| # | ラベル | 質問タイプ | 必須 | 備考 |
|---|---|---|---|---|
| 1 | `お名前 / Full Name` | 記述式（短文） | ✅ | フルネーム |
| 2 | `表示名 / Display Name` | 記述式（短文） | — | 紹介時の呼び方（空ならお名前を使用） |
| 3 | `国籍 / Nationality` | ラジオボタン | ✅ | 日本 / カンボジア / 中国 / 韓国 / その他 |
| 4 | `会社名 / Company` | 記述式（短文） | — | |
| 5 | `役職 / Title` | 記述式（短文） | — | 例: CEO / 代表 / 店長 |
| 6 | `コミュニティ・所属 / Community` | 記述式（短文） | — | 例: カンボジア日本人会、ロータリー |
| 7 | `電話番号 / Phone` | 記述式（短文） | ✅ | 国番号付推奨 |
| 8 | `Telegram Username（@なし） / Telegram Username` | 記述式（短文） | ✅ | @は不要（自動で除去） |
| 9 | `メールアドレス / Email` | 記述式（短文・メール検証） | — | |
| 10 | `ABA口座番号 / ABA Account Number` | 記述式（短文） | ✅ | コミッション送金先 |
| 11 | `口座名義 / Account Name` | 記述式（短文） | ✅ | パスポート通り |
| 12 | `希望する紹介コード（任意・4〜12文字英数字） / Preferred Referral Code (optional)` | 記述式（短文） | — | 例: YAMADA / TANAKA（`SM-` は自動で付与） |
| 13 | `どこで本プログラムを知りましたか？ / How did you hear about this?` | 記述式（短文） | — | 集客分析用 |
| 14 | `契約条件に同意します / I agree to the terms` | チェックボックス | ✅ | 選択肢「同意します / I agree」 |
| 15 | `連絡事項・ご要望 / Notes` | 記述式（段落） | — | 自由記述 |

#### 5-3. 「契約条件」セクションを前段に追加

質問14の前に**説明セクション**を挟み、以下の文面を貼り付け：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Samurai Motors パートナー契約条件
Samurai Motors Partner Terms
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 役割
パートナーは Samurai Motors の出張洗車サービスを知人・顧客に紹介します。

2. コミッション
紹介コード経由で成約した売上の 30% を、月末締め・翌月 ABA 送金します。

3. 帰属判定
お客さまが予約時に紹介コードを伝えた時点でパートナー実績として記録されます。
複数パートナーが関与した場合は、最初にコードを伝えたパートナーが対象となります。

4. 初回限定
コミッション対象はお客さまの「初回ご利用分のみ」です。

5. 支払
翌月10日までに ABA 口座へ送金。金額 $10未満の場合は翌月繰越とします。

6. 対象外
Samurai Motors の既存顧客（過去利用あり）の再紹介は対象外です。

7. 契約期間
承認日から 6ヶ月（以降、両者同意により自動更新）。

8. 税金
受け取るコミッションに係る税金はパートナー自身の責任で処理してください。

9. 守秘義務
受け取った顧客情報は第三者に開示しません。

10. 解約
両者は 30日前の通知で解約できます。Samurai Motors は違反時に即時解約できます。

11. 改訂
条件変更時は1ヶ月前に通知します。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ℹ️ 申込受付後、管理者が内容確認のうえ承認いたします。
   承認完了後、Telegram に紹介コード・体験コード等をお送りします。
```

上記の **カンボジア E-Commerce Law 2019 第48条** により、本フォームでの同意は電子契約として
有効です。紙の契約書は原則不要です。

#### 5-4. 「回答」タブ → スプレッドシートにリンク

設定タブ → 「回答先スプレッドシート」→ **v7-operations スプレッドシート** を選択
（または新しいシートを作って v7-ops と同じスプレッドシートに統合）。

※ 実装上は **onFormSubmit の event 経由** でデータを読むため、回答先シートはどこでも動作します。
ただし管理しやすさを優先し、v7-ops スプレッドシート内の「フォームの回答」タブに入れるのを推奨。

#### 5-5. フォーム ID を取得

フォーム編集 URL: `https://docs.google.com/forms/d/{FORM_ID}/edit`
→ `{FORM_ID}` 部分をコピー。

### STEP 6. onFormSubmit トリガーを登録

GAS エディタで以下を実行：

```javascript
setupPartnerFormTrigger('【STEP 5-5 でコピーしたフォームID】')
```

成功すれば「✅ handlePartnerFormSubmit の onFormSubmit トリガーを登録」ログが出ます。

### STEP 7. テスト送信

GAS で以下を実行すると、**フォーム回答を模擬した模擬投稿**が走ります：

```
debugMockPartnerFormSubmit()
```

確認ポイント：
- ✅ パートナーシートに「山田 太郎」の行が追加されている
- ✅ パートナーID が `PTR-YYMMDD-001` 形式
- ✅ 紹介コードが `SM-YAMADA`
- ✅ 体験コードが `SMTRY-YAMADA...`
- ✅ Telegram 管理グループ（パートナートピック）に承認ボタン付きメッセージが到着
- ✅ ✅承認 ボタンをタップ → 「✅ 承認済み by @xxx」に変化
- ✅ 承認後、パートナーシートのステータスが「承認済み」
- ✅ Chat ID 未登録のため Welcome DM は送信されず、「⚠️ Welcome DM 未送信」通知が届く

## 🔁 運用サイクル

### 申請→承認→運用のルーティン

1. **申請者**: 管理者から Google Form URL を受け取り回答
2. **管理者**: Telegram 通知の ✅承認 をタップ（却下の場合は ❌却下）
3. **承認後**: パートナーの Telegram Username に直接 DM して Chat ID を教えてもらう
   - 相手に「@SamuraiMotorsBot を友だち追加して /start を送ってください」と伝える
   - 相手から start が来たら、その Chat ID をパートナーシート「Telegram Chat ID」欄に貼り付け
   - `sendPartnerWelcomeDm_('PTR-XXXXXX-XXX')` を実行すると Welcome DM が飛ぶ
4. **紹介開始**: パートナーが紹介コード `SM-XXX` を知人に共有
5. **予約時**: お客さまが予約Bot に紹介コードを入力
6. **月末**: 紹介履歴から集計 → ABA 送金 → 支払ステータス「支払済み」に更新

### 承認後 Chat ID を取得するフロー

Google Form では Chat ID を取得できないため、**承認後に1度だけ手動連携が必要**です。

#### スマートな代替案（推奨）

承認時の Welcome DM が送れなかった場合、管理者に「@Username に直接送ってください」と案内が
出ます。この DM テンプレートをコピペで送るだけで OK：

```
🎉 Samurai Motors パートナー登録ありがとうございます！
以下の受け取り専用Botを起動してください：

👉 t.me/SamuraiMotorsInternalBot
Botを友だち追加し「/start」と送ってください。
送信後、紹介コード・体験コード・詳細な手引きが自動で届きます。
```

（注: Bot の Username は実際のものに置換）

## 🔐 セキュリティと権限

- Google Form の送信権限: **インターネット上の誰でも**（URL を知っている人だけ）
- 回答先スプレッドシートの閲覧権限: **管理者のみ**（編集権限は飯泉のみ推奨）
- パートナーの個人情報（電話・メール・口座）は v7-ops スプレッドシートに限定
- Welcome Kit PDF は必要に応じて「リンクを知っている全員」に設定

## ❓ よくある質問

### Q. 紹介コードを本人が決められる？
はい。Form の「希望する紹介コード」に入れれば採用されます。重複時は末尾に2桁の数字が自動付与されます。

### Q. 複数パートナーが同じお客さんを紹介した場合は？
先に紹介コードをお客さんに伝え、お客さんが予約時に入力したコードのパートナーが実績となります（first-code-wins）。

### Q. フォーム送信後、すぐ承認通知が来ない場合は？
Google Form のトリガーには 0〜10秒の遅延があります。5分待っても来ない場合：
1. `setupPartnerFormTrigger(formId)` を再実行
2. GAS エディタ → 実行ログ を確認

### Q. 申請者が間違って2回送信した場合は？
パートナーIDは自動で連番採番されるので2件登録されます。重複に気づいた時点で古い方を手動で「却下」にし、備考に「重複申請のため却下」と記載してください。

### Q. 契約終了日を過ぎたら？
現状は自動停止しません。シートの「ステータス」を「停止」に手動変更してください。将来的には月次トリガーで自動化予定。

## 📞 困ったら

管理者チームに Telegram で連絡してください。フォーム・GAS・スプレッドシートの異常は
v7-operations のログを確認することで大抵解決できます。
